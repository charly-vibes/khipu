## Project-Specific AI Context
_Last reflected: 2026-03-03 · 0 sessions analyzed_

### Architecture Notes
- Pipeline is strictly linear: ingest → redact → condense → analyze → emit. Each stage is independent and configured via files, not code.
- Three interfaces (CLI, MCP, library) share the same core. Entry points: `cli.py` (typer), `mcp_server.py`, `__init__.py` (public API).
- Ingestors live in `ingestors/` (built-in) or `.khipu/ingestors/` (user-local) or `~/.config/khipu/ingestors/` (global). Auto-discovered, no registration.
- Analyzers are prompt `.md` files with YAML frontmatter (`id`, `version`, `depends_on`). They form a DAG — run in dependency order. `crystallize` depends on `patterns`.
- Emitters are Jinja2 `.j2` files. Prompt variables use `{variable}` (simple substitution); emitter templates use `{{ variable }}` (Jinja2). Do not confuse them.
- Backends are TOML files. Two modes: `api` (HTTP) or `cli` (shell out). Any program that reads stdin/file and writes stdout is a valid backend.
- Config discovery: project (`.khipu/`) > user (`~/.config/khipu/`) > built-in. Same-id files: project wins.

### Conventions
- Ingestors define `can_handle(path)` + `ingest(path)`. Optionally declare `EXTENSIONS`, `FILENAMES`, `CONTENT_PATTERN`, `PRIORITY` constants for fast two-phase detection.
- `generic` ingestor always has `PRIORITY = -100` (last resort). Specific ingestors use `PRIORITY = 10`.
- Redaction is on by default. Never suggest `--no-redact` for external/API backends.
- Condensation default is `auto`: estimate tokens (`len(json_text) / 4`), reserve 30% for prompt+response, condense if sessions exceed 70% of context_limit.
- JSON extraction from LLM responses: strip markdown fences, strip preamble before first `[`/`{`, strip postamble after last `]`/`}`, parse, retry once on failure.
- `Session.source` values: `"claude-code"`, `"cursor"`, `"aider"`, etc. (lowercase, hyphenated).

### Common Gotchas
- Stdin input (`-`) requires `--ingestor` flag — format detection cannot work without a path.
- Prompt files use `{variable}` substitution, NOT Jinja2. Emitters use `{{ variable }}`. Mixing them up breaks rendering silently.
- `crystallize` analyzer depends on `patterns` — both must be in the analyzer list or `crystallize` will error at load time.
- When sessions exceed context limit even after condensation, khipu takes the most recent that fit (not an error — just a warning + skip).
- `_condense.md` (underscore prefix) is the smart condensation system prompt — not user-facing, not listed by `khipu prompts`, cannot be overridden.
